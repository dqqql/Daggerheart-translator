# 匕首之心翻译参考

**不是术语表**——所有 1:1 的译名见 `terms-14448.json`。
这里只放术语表解决不了的：上下文相关选择、格式规范、句式模板。

## 点 / 值 / 槽 分类原则

- **点**：频繁增减的资源。希望点、恐惧点、压力点、生命点。
- **值**：一般不动的数值。闪避值、护甲值、熟练值。
- **槽**：容量上限。希望槽、生命槽、压力槽。

唯一的例外是**护甲槽**——它是频繁增减的资源，但因为英文规则里用"armor slot"，所以叫"槽"而不是"护甲点"。

## Clear的四种译法

| 宾语 | 中文 | 原因 |
|------|------|------|
| 生命点 | **恢复** | "清除生命点"在中文里有歧义, 容易误解为受伤, 实际是恢复 |
| 压力点、护甲槽 | **清除** | 擦掉铅笔标记 |
| 指示物 (Token) | **移除** | 与"清除压力点"区分，桌游惯例 |
| 恐惧点 (GM 侧) | **清除** | GM 自己消掉恐惧池里的点 |

## 资源变动

资源的变动有一套统一的术语: `[动词] [数量] [名词]`
- `[动词]`: 标记(mark)/清除(clear)/移除(remove)/花费(spend)/获得(gain)/失去(lose)/...etc
- `[数量]`: 1~10/1d4/一或更多/至少1/等量的/...etc
- `[名词]`: 生命点|希望点|压力点|恐惧点/...etc

最终形式为: '标记 1 生命点', '花费 等量的 希望点'. 
在makeup.py中会用正则处理大多数此类短语, 但是不排除一些特殊行文或格式导致匹配失败. 此时需要你手动按照这个固定格式修正它

## 攻击范围

- 将其击退至 XX 范围**处** → 必然落在该范围最远处
- 移动到 XX 范围**内** → 范围内任意选位置
- **额外攻击范围内的另一名敌人** → 同一次攻击掷骰，再加打一个
- **对 XX 范围内的所有其他目标使用这次攻击** → 同一次掷骰，打范围内全部
- **命中攻击范围内的一个额外目标** → 不需要掷骰，直接命中

## 共创叙事句式

- **游戏主持人会告诉你**……
- 向游戏主持人**提出等量下列问题**
- **与游戏主持人一起**确定……

## 名称原文标注

对于以下专有名词，需在译名后附带英文原名，格式为 `{译名} {原名}`。

- **游戏机制名词**: 敌人/环境/领域/领域卡/职业/子职/社群/种族
- **已有译名的作品**: 灵感来源中的电影,游戏名等, 正文中提及的其他作品等

格式：`火山巨龙：暴君余烬 VOLCANIC DRAGON：ASHEN TYRANT`

简称规则不变：同一敌人/领域卡后续引用时可只用简称，不带英文原名。

## 7 级领域卡 xx-Touched 翻译

7 级领域卡中，名为 `xx-Touched` 的卡牌统一译为 `{xx译名}恩泽`，其中 `xx` 为对应领域的英文名。

示例：
- `Blade-Touched` → `利刃恩泽`（领域 Blade = 利刃）
- `Bone-Touched` → `骸骨恩泽`（领域 Bone = 骸骨）
- `Grace-Touched` → `优雅恩泽`（领域 Grace = 优雅）

注意：`xx` 的译名严格复用术语表中该领域的已有译法，不得自创。
